轮播图组件Swiper

根据官方API

swiper 是滑块视图容器 ，只可放置swiper-item组件 。

swiper-item : 宽高会自动设置为100%。

其属性在v1.9.0版有：

  • item-id ：string 型， 为该项标识符

  • skip-hidden-item-layout ：boolean型，是否跳过未显示的滑块布局

通过给swiper 设置其属性 如 autoplay: true; 就会自动播放。

so, 使用方式就是通过给各个标签设置 class名 , 用样式来控制显示形式。

用属性控制行为，用事件控制变化等。

v1.0.0版 部分属性如下：

indicator-dots [false] ; 是否显示面板指示点

autoplay [false] ; 是否自动切换

interval [5000] ; 自动切换时间间隔

duration [500] ; 滑动动画时长

current [0 : number] ; 当前所在滑块的 index

scroll-view 组件

官方文档

注： 官方示例代码中，三个展示view 样式 背景色等是写在 app.wxss中， 且用的伪元素放置文字ABC

可滚动视图区域。使用竖向滚动时，需要给scroll-view一个固定高度，通过 WXSS 设置 height。组件属性的长度单位默认为px，2.4.0起支持传入单位(rpx/px)。

其属性：

scroll-x [false] ; 允许横向滚动

scroll-y [false] ; 允许纵向滚动

scroll-top [number/string] ; 设置竖向滚动条位置

scroll-left [number/string] ; 设置横向滚动条位置

scroll-into-view [string] ; 值应为某子元素id（id不能以数字开头）。设置哪个方向可滚动，则在哪个方向滚动到该元素 ; 优先级高于scroll-top

bindscrolltoupper [eventhandle] ; 滚动到顶部/左边时触发

bindscrolltolower [eventhandle] ; 滚动到底部/右边时触发

  1. [渲染层网络层错误] Failed to load font https://at.alicdn.com/t/font_2553510_iv4v8nulyz.woff2?t=1649083952952

    net::ERR_CACHE_MISS

解决办法： -文

原因：异步请求需要一定的时间，而小程序一进页面就开始加载，请求速度没跟上，这时页面显示加载的时候image里面的值是空的，就会出现渲染层、网络层错误，

解决：

只需要加上wx:if="{{arr.length>0}}" 或者wx:if="{****}}"
拖延一下加载图片的时间

<image wx:if="{{ item.imgSrc }}" src="{{ item.imgSrc }}"
