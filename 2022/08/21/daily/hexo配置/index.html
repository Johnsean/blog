hexo 配置教程：

命令：

文件：

hexo new “post title with whitespace” //标题包含空格，用引号括起来

-p //自定义文章路径

-r // 若存在同名文章 则替换

-s // 文章的 Slug, 作为新文章的文件名和发布后的URL

默认情况下，Hexo 会使用文章的 title 来决定文章文件的 path。对于独立页面来说，Hexo 会创建一个以title 为名字的目录，并在目录中放置一个 index.md 文件。你可以使用 --path 参数来覆盖上述行为、自行决定文件的目录：

​ - 创建一个 source/about/me.md 文件，同时 Front Matter 中的 title 为 "About me"

​ - 创建 source/_posts/about/me.md，同时 me.md 的 Front Matter 中的 title 为 "page"

其他

hexo g // 生成静态html 文件

hexo s //开启服务器 -p 指定端口 -i 指定ip -s 静态模式

hexo d // 部署到线上版本库，触发 xx.yml中的部署任务

hexo clean // 清除静态Html 文件目录

hexo –debug //调式模式

头部（post）

配置项（可选）：

copyright : false 转载文章, 不需要显示版权

indexing: false 排除被搜索

sticky 置顶。数字越大，置顶的优先级越大

tags 文章标签
categories 分类
updated 更新日期 , 默认值/不写 为当天；
keywords 文章关键字
top_img 顶部图片
toc 目录
aside 显示侧边栏 (默认 true)
aplayer 加载aplayer的js和css,请参考 音乐 配置
highlight_shrink 代码框是否展开(true/false) 默认false
comments 显示页面评论模块(默认 true)
mathjax 数学式 beauty
katex 公式 beauty
